

## 1. 产品概述

​耄耋记账是一款面向个人用户的智能记账应用，旨在帮助用户轻松管理日常收支、分析消费习惯、制定预算计划，实现财务健康管理。

### 产品定位

- ​**目标用户**​：有记账需求的个人用户，特别是年轻上班族、学生群体，结合耄耋无版权形象引流
- ​**核心价值**​：简单高效的记账体验 + 智能财务分析 + 个性化预算管理 + 耄耋的陪伴
- ​**差异化优势**​：智能分类 + 多场景记账 + 纯净化
    - 利用快捷服务和全局弹窗实现快速记账
    - 基于图像识别实现自动填充与分类
    - 接入deepseek实现AI耄耋
    - 数据可视化能不能基于复杂算法进行优化，类似于js-d3
    - 要不要把未来计划做进去

## 5. 技术实现方案

抉择spring和fastapi

### 关键技术点

1. ​**AI分类引擎**​
    - 采用BERT+BiLSTM混合模型
    - 本地轻量化模型（<10MB）
    - 增量学习更新机制

2. ​**数据同步方案**​
    - 差分同步算法减少流量消耗
    - 冲突解决策略：时间戳+用户确认
    - 端到端加密传输（AES-256）
3. ​**性能优化**​
    - 本地优先策略
    - 数据分页加载
    - 后台数据预取

### 技术栈选择

| 模块          | 技术方案                           | 选型理由      |
| ----------- | ------------------------------ | --------- |
| ​**跨平台UI**​ | Compose                        | Android优先 |
| ​**后端服务**​  | Spring/fastapi                 | 高并发/生态丰富  |
| ​**数据库**​   | SQLite(本地)+MongoDB(云)          | 轻量+扩展性    |
| ​**AI引擎**​  | TensorFlow Lite + Scikit-learn | 移动端优化     |
| ​**数据同步**​  | MQTT协议                         | 低功耗/实时性好  |

## 8. 风险与应对

### 主要风险矩阵

|风险类型|可能性|影响|应对措施|
|---|---|---|---|
|数据安全风险|高|极高|端到端加密/本地存储优先/GDPR合规|
|用户增长缓慢|中|高|裂变机制/场景合作/免费增值策略|
|分类准确率不足|高|中|人工修正反馈机制/多模型融合|
|盈利模式单一|中|中|多元化变现/企业服务拓展|
|竞争加剧|高|高|持续创新/细分场景深耕|

### 数据安全方案

1. ​**加密策略**​：
    
    - 传输层：TLS 1.3 + AES-256
    - 存储层：SQLCipher加密数据库
    - 认证：生物识别+二次验证
2. ​**隐私保护**​：
    
    - 数据最小化原则
    - 用户数据所有权明确
    - 匿名化数据分析
3. ​**合规性**​：
    
    - GDPR/CCPA合规
    - 等保三级认证
    - 定期安全审计
